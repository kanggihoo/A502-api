### 최종 전체 카테고리 분석 종합 정리

지금까지 확인한 전체 GitLab API 카테고리를 최종 정리하면 다음과 같습니다.

#### 1. 필수 활용 대상 (P0 - 핵심 기능)
- **`162-users`**: 로그인 사용자 식별, 프로필, 소속 그룹/프로젝트 조회 (`28, 03, 05, 26번`).
- **`129-projects`**: 대시보드 기준 엔터티, 팀 저장소 자동 생성 (`12, 13, 16, 28, 64~67번`).
- **`67-groups`**: 팀/반 그룹 구조 식별, 프로젝트 자동 탐색 (`11, 12, 19, 22, 29, 31번`).
- **`91-members`**: 팀원 목록, 역할(`Maintainer`/`Developer`) 판별 및 팀원 추가 (`03, 09, 10, 14번`).
- **`93-merge-requests`**: 리뷰 대기 목록, MR 상세, 커밋, 변경 사항, 병합 (`09, 10, 16, 19, 21~23, 27~29, 32, 35, 36번`).
- **`35-commit-statuses`**: **젠킨스 빌드 상태를 GitLab 커밋/MR에 전송/반영 (`01, 02번`)**.
- **`36-commits`**: 커밋 활동 이력, 코드 변경 내역, MR 연동 (`01, 03, 04, 11번`).
- **`52-events`**: 팀원 활동 타임라인 및 대시보드 실시간 피드 (`01, 02, 03번`).
- **`68-hooks`**: **Push, MR, 이슈 이벤트를 수신하여 Mattermost 실시간 알림 전송 (`27~31, 33번`)**.
- **`138-repositories`**: 저장소 트라이 탐색 및 `develop` vs `main` 브랜치 비교 (`01, 05, 07번`).

---

#### 2. 선택/보조 활용 대상 (P1/P2 - 부가 기능)
- **`136-releases`**: 릴리스 버전 및 배포 히스토리 모니터링 (`06, 07, 09번`).
- **`49-environments` & `45-deploy-resources`**: EC2 배포 타겟 상태 및 배포 MR 연동 (`02, 07, 16번`).
- **`131-protected-branches`**: 기본 브랜치 보호 상태 점검 및 초기 설정 (`01, 02번`).
- **`145-search`**: 통합 대시보드 내 프로젝트/그룹 자원 검색 (`04, 05, 06번`).
- **`139-resource-events`**: 이슈/MR 상태 변경 및 라벨 타임라인 추적 (`05~08, 11~14, 17~20번`).
- **`92-merge-request-approvals`**: MR 승인 상태 확인 및 대시보드 승인 처리 (`10, 12번`).
- **`154-suggestions`**: MR 인라인 코드 리뷰 제안 즉시 커밋 반영 (`01, 02번`).
- **`156-templates`**: 프로젝트 생성 시 공통 `.gitignore`, `Dockerfile` 제공 (`03, 04, 07, 08번`).
- **`71-integrations`**: 젠킨스/Jira/Mattermost 내장 연동 자동화 (`25, 26, 43번`).
- **`57-files`**: README.md 등 대시보드 원문 미리보기 (`03, 05번`).
- **`89-markdown`**: GFM Markdown 렌더링 (`01번`).
- **`95-metadata`**: `lab.ssafy.com` 서버 버전 확인 (`01, 02번`).
